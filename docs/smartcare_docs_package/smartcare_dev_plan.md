# SmartCare 智慧社区养老监护系统
## 项目方案与阶段开发计划（开发人员版）

---

## 1. 项目目标

SmartCare 是一个基于 AI 视觉识别的社区养老安全监护系统。
系统通过摄像头实时分析老人行为，自动识别跌倒或异常行为，并将事件发送到后端系统进行记录与展示。

核心目标：
- 自动识别跌倒等异常行为
- 实时生成安全事件
- 提供管理平台查看告警与历史

---

## 2. MVP 范围

当前阶段实现最小可行系统（MVP）：
1. AI 行为识别（以跌倒检测为主）
2. 事件接收与记录服务
3. 告警记录管理
4. Web 管理页面展示

---

## 3. 系统整体架构

模块：
1. AI 识别模块（Python）
2. 事件服务模块（Spring Boot）
3. 数据存储模块（MySQL）
4. 管理平台模块（Web）

流程：
摄像头 -> AI 模块 -> Spring Boot 事件服务 -> MySQL -> Web 管理平台

---

## 4. 模块设计

### 4.1 AI 识别模块
职责：
- 接收视频流
- 识别人体与姿态
- 检测跌倒
- 生成事件数据

示例事件：
```json
{
  "elder_id": 1,
  "event_type": "fall",
  "timestamp": "2026-03-07T10:20:00"
}
```

技术建议：Python + OpenCV + YOLO + MediaPipe

### 4.2 事件服务模块（Spring Boot）
职责：
- 接收 AI 事件
- 校验请求数据
- 写入数据库
- 提供 REST API 给前端

主要接口：
- `POST /api/events`
- `GET /api/alerts`
- `GET /api/elders`

技术建议：Java 17 + Spring Boot + Spring Web + Spring Data JPA

### 4.3 数据存储模块
存储内容：
- 老人信息
- 告警记录
- 行为事件记录

数据库：MySQL

主要表：
- `elders`
- `alerts`
- `events`

### 4.4 管理平台模块
功能：
- 老人信息管理
- 告警记录查看
- 实时状态展示
- 基础统计展示

## 5. API 设计（初版）

### 接收 AI 事件
`POST /api/events`

请求示例：
```json
{
  "elder_id": 1,
  "event_type": "fall",
  "timestamp": "2026-03-07T10:20:00"
}
```

### 获取告警记录
`GET /api/alerts`

返回示例：
```json
[
  {
    "id": 1,
    "type": "fall",
    "time": "2026-03-07 10:20"
  }
]
```

## 6. 阶段计划

### 第1周：架构与环境
- 明确模块边界
- 初始化仓库与目录

### 第5周：数据库与数据管理
- 建表与索引
- 完善读写接口

### 第6周：Web 管理端
- 展示告警、老人信息
